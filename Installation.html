Cloud Trader Pro

Enterprise-Grade Algorithmic Trading Infrastructure Installation & Deployment Guide

System Architecture Overview

Cloud Trader Pro operates under a robust, Headless Decoupled Architecture which segregates core trading mechanisms from standard user-facing controllers:

  • Headless Backend (Server): A background engine running autonomously. It maintains persistent broker links, handles live web socket market ticks, and runs algo strategies.
  • Web Dashboard (Remote): A modern browser portal served by the backend to manage credentials and track status remotely.
  • Tkinter Desktop UI (Local): A specialized desktop console for professional scalping and detailed configuration connecting directly to the server backend.

Option 1: Docker Deployment (Recommended)

Using Docker encapsulates the application, libraries, and runtime environment. It eliminates the need to manually install Python or configure dependencies on the host system.

1. Install Docker on Ubuntu Linux:

Execute the following commands to install Docker Engine and Docker Compose:

bash 
# Update package list and install prerequisites
sudo apt update && sudo apt upgrade -y
sudo apt install -y curl bzip2 tar git

# Install Docker Engine and the Compose plugin
sudo apt install -y docker.io docker-compose-v2

# Start and enable the service to boot automatically
sudo systemctl enable --now docker

1. Install Docker Desktop on Windows:

  • Download the official installer: Docker Desktop for Windows.
  • Run the installer and make sure that the option to enable the WSL 2 backend is checked.
  • Restart your computer if prompted by the installation wizard.

2. Prepare Deployment Directory

Create a dedicated folder and move the deployment assets inside it:

bash
sudo mkdir -p /opt/cloudtraderpro
cd /opt/cloudtraderpro
powershell
mkdir C:\CloudTraderPro
cd C:\CloudTraderPro

Copy the following configuration assets from the release package to this folder:

  • docker-compose.yml
  • cloudflare_tunnel_command.txt
  • setup.sh (for Linux) or setup.bat / setup.ps1 (for Windows)
  • The configs/ directory containing settings samples.
  • The cloudtraderpro_v1.tar file (if deploying a portable release).

3. Configure App Settings

Go to the configs/ folder, rename app_settings.json.sample to app_settings.json, and configure your parameters:

json 
{
    "dashboard_password": "your_secure_web_password",
    "local_api_key": "shared_secret_for_tkinter_app",
    "active_broker": "flattrade", 
    "jwt_secret": "long_random_string_for_web_tokens",
    "session_timeout_minutes": 86400,
    "server_port": 8002,
    "license_key": "your_license_key"
}
Important Notice

Always customize the default dashboard password, local api key, and jwt secret. The license_key field is required to cryptographically validate your software license upon server startup.

4. Boot and Manage the Server

If using a portable tarball file, load the image into Docker first:

bash
sudo docker load -i cloudtraderpro_v1.tar
sudo docker compose up -d
powershell
docker load -i cloudtraderpro_v1.tar
docker compose up -d

5. Stopping & Removing Containers

If you need to stop the server or clean up container resources, select your OS for the appropriate commands:

Stop the running containers (keeps data intact):

bash
sudo docker compose stop

Stop and completely remove containers (frees up ports & network):

bash
sudo docker compose down

Remove containers and delete loaded image (to free up disk space):

bash
sudo docker compose down
sudo docker rmi cloudtraderpro:1.0

Stop the running containers (keeps data intact):

powershell
docker compose stop

Stop and completely remove containers (frees up ports & network):

powershell
docker compose down

Remove containers and delete loaded image (to free up disk space):

powershell
docker compose down
docker rmi cloudtraderpro:1.0

Step 5: Updating the Docker Container (When a New Version is Released)

When a new version of the Cloud Trader Pro image is uploaded to Docker Hub, you can update your setup easily without losing your configurations, databases, or logs (since they are stored outside the container in your host directories).

Navigate to your deployment directory and run the standard update sequence:

bash 
cd /opt/cloudtraderpro
sudo docker compose pull
sudo docker compose up -d

Open PowerShell, navigate to your deployment directory, and pull/start the new container:

powershell 
cd C:\CloudTraderPro
docker compose pull
docker compose up -d
Linux Command Chaining Tip

If you choose to chain commands together with && on Linux, remember that sudo only applies to the command immediately following it. You must prepend sudo to both parts of the command line, for example:
sudo docker compose down && sudo docker compose up -d
Omitting the second sudo will trigger a Permission Denied socket error.

How it works & Custom Port Persistence:
  • Local Configurations Safe: Your API keys, passwords, and other credentials inside configs/app_settings.json are stored on the host machine and are never modified or overwritten by image updates.
  • Port Settings Remain Intact: If you mapped a custom host port (e.g. 8003:8002) in your local docker-compose.yml, that file will not be changed. Your custom port mapping will continue to function on the new container version.
  • Caution with Installer Script: If you update by re-running the full installer script (install.sh/install.ps1), it will download a fresh copy of docker-compose.yml from GitHub, which will overwrite your custom port settings. If you do this, you will need to re-apply your port mapping changes in docker-compose.yml.

Option 2: Local Python setup

If you wish to bypass Docker containerization and run Python directly, follow these steps.

Prerequisites:

  • Install Python 3.12+
  • Install the uv package manager (recommended for fast dependencies synchronization).

1. Install `uv` Package Manager:

bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env
powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

2. Install Tkinter & Dependencies:

On Linux systems, python's built-in Tkinter package must be installed globally:

bash
sudo apt update
sudo apt install -y python3-tk

3. Sync Packages & Boot Server:

Navigate to the project root and run:

bash
# Synchronize environment excluding development utilities
uv sync --no-dev

# Run backend server
uv run python run_server.py

Troubleshooting & Diagnostics

"Illegal Instruction" / NumPy Compatibility Crash

Recent compiled packages for NumPy and Pandas require advanced CPU capabilities (like AVX instruction sets). On older CPUs, this triggers compatibility crashes.

Solution: Since end-users cannot rebuild the prepackaged Docker image themselves, you must run the server using Local Python Deployment (Option 2), and force older, compatible package versions using:

bash
uv add "numpy<2.0" "pandas<2.3"

SSL Certificate Validation Errors (Windows)

If the Windows host machine fails to fetch symbols or connect to broker APIs due to SSL/TLS handshake exceptions, the local Certificate Authority store may be outdated.

Solution: Run PowerShell with Administrative Privileges and run these commands to update root certificates:

powershell
certutil -generateSSTFromWU roots.sst
certutil -addstore -f root roots.sst
del roots.sst

Docker Container Inspection

Inspect the state and console logging of your running docker services:

docker cli
# Track live server stdout/stderr stream
docker compose logs -f

# Jump into container CLI shell for directory analysis
docker exec -it cloud-trader-pro bash

Networking & Cloudflare Tunnel

Exposing the local FastAPI endpoint to the public internet securely (with SSL/TLS) is essential for receiving broker callbacks. A Cloudflare Zero Trust Tunnel is the recommended way to set this up.

1. Obtain Tunnel Token:

  • Access your Cloudflare Zero Trust Console.
  • Go to Access -> Tunnels -> Create a Tunnel.
  • Under the "Docker" configuration guide, copy the full docker run... command (or copy just the raw token string).

2. Run Automatic Setup helper:

Open cloudflare_tunnel_command.txt in a text editor and paste the copied token or the entire command. Save the file and run the installer script:

bash
chmod +x setup.sh
sudo ./setup.sh
powershell
# Double-click setup.bat or run:
.\setup.ps1

The helper automatically updates the .env file, establishes a docker-compose.override.yml containing the Cloudflare Tunnel container sidecar, and initiates the service stack.

3. Configure Route Hostname:

  • In the Cloudflare dashboard, edit your active tunnel and go to the Public Hostname tab.
  • Click Add a public hostname and set:
    • Subdomain: e.g., trader
    • Domain: Your registered domain (e.g., domain.com)
    • Type: HTTP
    • URL: cloud-trader-pro:8002
  • Save settings. Now, your server dashboard is publicly available over secure HTTPS at https://trader.domain.com.
  • Broker Portal Bindings: Set your broker redirect Callback URI to: https://trader.domain.com/api/auth/callback

Tkinter Desktop Client Setup

The Tkinter desktop application (main.py) is designed to run on a local machine (Windows PC or local Linux Desktop). It cannot display a graphical interface if run directly on a headless Ubuntu Linux Server without a desktop environment; however, it can connect remotely to a backend hosted on an Ubuntu Server.

Development Status

A compiled standalone Windows .exe package is planned for future release. Currently, the Tkinter app is executed using Python. Both the Python execution method and the future standalone .exe require proper connection secrets to communicate with the server backend.

Starting the Desktop Client:

Navigate to the project root directory on your local machine and run:

bash
uv run python main.py

Configuring connection secrets (`configs/cf_secrets.json`):

Create or edit the configs/cf_secrets.json file in your local application directory to configure the remote server coordinates and bypass Cloudflare Access policies:

json
{
    "base_url": "https://trader.yourdomain.com",
    "ws_url": "wss://trader.yourdomain.com/ws",
    "use_cloudflare": true,
    "local_api_key": "must_match_app_settings_on_server",
    "CF-Access-Client-Id": "your-cloudflare-client-id.access",
    "CF-Access-Client-Secret": "your-cloudflare-client-secret-hex-string"
}
  • base_url & ws_url: The connection endpoints pointing to your server:
    • Local Python setup / SSH Port Forwarding: Use "base_url": "http://localhost:8002" and "ws_url": "ws://localhost:8002/ws" (with "use_cloudflare": false).
    • Cloudflare Tunnel Remote setup: Use "base_url": "https://trader.yourdomain.com" and "ws_url": "wss://trader.yourdomain.com/ws" (with "use_cloudflare": true).
  • local_api_key: Must match the local_api_key value in the remote server's app_settings.json.
  • CF-Access-Client-Id & CF-Access-Client-Secret: If you are protecting your domain via a Cloudflare Access Policy (SSO), you must configure a **Service Token** in your Cloudflare Zero Trust Console and paste the resulting credentials here. This allows the Tkinter client to authenticate cryptographically and bypass the browser login gate.
  • use_cloudflare: Set to true to pass the Access headers, or false if connecting locally via localhost (e.g. over SSH Port Forwarding).

Alternative Access Methods (Without a Domain)

If you do not own a custom domain, you can deploy the app using one of these three options:

Option A: Cloudflare Quick Tunnels (Random Temporary Domain)

Run a temporary public tunnel without creating an account. Add this service block to your docker-compose.yml file:

yaml
quick-tunnel:
  image: cloudflare/cloudflared:latest
  command: tunnel --url http://cloud-trader-pro:8002

Boot up the container and check the container log to locate the generated URL link:

bash
docker compose logs quick-tunnel

Look for a line containing: https://some-random-words.trycloudflare.com.

Option B: SSH Port Forwarding (Recommended for Private Access)

Keep your firewall ports locked and secure. Establish an encrypted tunnel from your local PC to the remote server by running:

bash
ssh -N -L 8002:localhost:8002 user@YOUR_SERVER_IP

Now, open http://localhost:8002 in your local browser to access the server safely.

Option C: Request a Subdomain from the Developer/Admin (Easiest)

If you do not own a domain, you can request the developer/admin to provision a custom subdomain (e.g. yourname.shoonyatrader.in) on their own network infrastructure. They can link this subdomain directly to your Cloudflare Tunnel connector.
